# Security Policy

We take the security of IgnisAI seriously. If you believe you have found a security vulnerability or threat signature escape in the platform, please report it to us responsibly using the instructions below.

---

## Supported Versions

Only the latest release version of the platform receives active security updates:

| Version | Supported |
| :--- | :---: |
| v1.0.x |  Yes |
| < v1.0.0 |  No |

---

## Reporting a Vulnerability

**Do NOT create a public issue on GitHub for security vulnerabilities.**

Instead, please report vulnerabilities via email or open an encrypted communication draft. Include:
1. **Description**: The nature of the exploit (e.g., threat detection bypass, JWT verification escape, role-escalation path).
2. **Steps to Reproduce**: Detailed steps or payload scripts (e.g., sample cURL requests or Pytest scenarios).
3. **Impact**: Potential consequences if exploited in production setups.

We aim to acknowledge reports within 48 hours and provide a resolved patch release within 14 days of verification.
